# DailyGate — the agent that earns trust

An **autonomous team-management agent** that runs across GitHub / Slack / email.
Autonomy is the default; human input is the exception — **and the line between
"act alone" and "ask first" moves as the agent earns the manager's trust.**

> "Most agents either ask permission for everything (annoying) or act on everything
> (terrifying). Ours starts cautious and *earns* autonomy — it learns which
> decisions you trust it to make alone, and which you always want to weigh in on."

## How it works
```
GitHub/Slack event ──Guild trigger──▶ Agent (TS, in Guild)
  ├─ context()   → ClickHouse: work, workload, trust[category]
  ├─ DECIDE:  stakes HIGH & irreversible → ALWAYS escalate (ceiling)
  │           trust=auto & confident     → ACT
  │           else                       → escalate
  ├─ ACT      → Composio (assign / close dup / email / Slack)
  ├─ ESCALATE → Guild ui_prompt (manager approves/overrides)
  └─ decision() → ClickHouse: record + recompute trust (3 approvals → auto)
  traced → Langfuse · supervised → OpenUI console
```

## Repo layout
```
contract/   🔒 frozen seam — types, schema, fake fixture (shared by both lanes)
agent/      Person A — the Guild agent (generated by `guild agent init`) + tools/policy
ui/         Person A — manager console (autonomy feed, escalation queue, trust dashboard)
data/       Person B — ClickHouse schema, seed, queries, /context + /decision API  (see data/PLAN.md)
```

## Sponsor stack
**Guild** (runtime + triggers + ui_prompt escalation) · **Composio** (read/act toolkits) ·
**ClickHouse** (workload + trust ledger) · **Langfuse** (traces the decisions) ·
**OpenUI/Thesys** (the supervision console).

## Run (once wired)
```bash
# agent  — after `guild auth login`
cd agent && npx -y @guildai/cli@0.12.3 agent test
# data   — Person B's API (default :8000)
# ui
cd ui && npm install && npm run dev   # → http://localhost:5173
```

Frozen constants: **N=3 approvals → auto · override → ask · ceiling never promotes · load>70 overloaded.**
